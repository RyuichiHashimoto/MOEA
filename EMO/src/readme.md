はじめに
--------
	EMOアルゴリズムをまとめたもの．
	合わせ実験等でプログラムの精度は保障されているが，利用する際は確認をしてください
	見たらわかるかと思いますが基本はJmetalをベースに書いています

設計思想
--------
	
	外部からソースコードを持ってきやすいように，Jmetalをベースに書いている．基本的に実行速度が最適化で保守性のあるプログラムを組むよう心かげている．
	基本的に一つのCPUで一回試行をこなすことをメインに考えている．基本は並列実装はほどこしていない．しかし，SMS-EMOAシリーズのアルゴリズムは計算量を複数のCPUで分散させるため，並列化を行うバージョンも実装している．具体的には，並列バージョンを施している．
	
	そして，各試行の並列化は実行せずにスクリプト(主にpython)で並行実行することによりCPUを無駄なく使用するよう心掛けている．
	c++ではなくjavaを選択したのはlinuxでも回せるようにするためだ
	スクリプト言語としてpythonを選んだのもそのため(個人的にシェルやバッチファイルは苦手)
	
	
実行方法
--------

	1. gritComputing を使用するケース
		大量に実験を回す時に使用．GridComputingの知識が少し必要
	
	2. それを使用しないケース．
		一つのみの実験を回す時に使用．どっかにある並列実行プログラムも必要
	
-ソースコードの説明

	--core:プログラムの基盤となるクラス
	
	--experiments:実験を回すクラスと実験設定を取り扱うクラス　experiments.startを指定して回すことを推奨します．コマンドライン入力と，ファイル読み込みにより実験設定を指定する．
		
	--metaheuristics: 多目的最適化遺伝的アルゴリズム．NSGAIIIは合わせれていないのであまり参考にしないでください
	
	--operators:交叉，突然変異，選択
	
	--problem:多目的ベンチマーク問題
	
	--utils：そのたもろもろ，結構大事なクラスもある
